## 2. 暂定题目

面向日频模拟交易的多 Agent 股票决策支持系统设计与实现

## 3. 问题描述与使用场景

股票分析和模拟交易决策通常需要综合多种异构信息，包括历史价格走势、技术指标、财经新闻、公司基本面以及风险约束。这些信息来源分散、形式不一致，并且不同来源之间可能互相冲突，导致人工整理成本高、决策过程不透明。

本项目拟构建一个基于大语言模型和多 Agent 协作的股票决策支持系统，面向日频模拟交易场景，为用户提供结构化的买入、卖出、持有或调仓建议。系统不接入真实券商账户，不进行实盘自动执行，而是在历史数据或离线样例环境中运行，重点服务于投资研究和课程实验。

为保证项目范围清晰，系统任务限定为三类：

1. 单股分析：给定某支股票近期价格、新闻和基本面信息，输出交易建议、支持理由和风险提示。
2. 候选股票筛选：给定一个小型股票池，识别相对值得关注或需要规避的标的。
3. 小型组合调仓：在给定持仓和风险约束下，生成调仓建议并说明依据。

项目目标不是构建自动炒股系统，而是实现一个可解释、可验证、可交互的决策支持原型。

## 4. 选择路线

Applied Track。

本项目的主要贡献在于构建可运行的软件系统，围绕股票分析这一具体场景，设计多模块协作流程、信息组织方式、风险提示机制和用户可解释输出。项目重点是系统设计与应用验证，而不是提出新的金融理论。

## 5. 核心系统设计

系统采用多 Agent 协作方式，而不是由单一 Agent 直接读取全部输入并输出结论。初步模块包括：

- 市场信息 Agent：读取历史价格和成交量，识别趋势、波动和短期异常。
- 新闻与情绪 Agent：处理财经新闻标题、摘要或事件信息，提取潜在利好、利空和事件驱动信号。
- 基本面分析 Agent：读取公司公开财务指标，对公司经营状态进行总结。
- 风险管理 Agent：检查持仓上限、集中度、回撤风险、现金比例和交易限制。
- 决策 Agent：综合各模块输出，给出买入、卖出、持有或调仓建议。
- LLM 复核 Agent：读取结构化中间结果，对最终结论进行保守复核和语言解释。

系统会展示各模块的中间分析结果，而不只是给出最终结论。

## 6. 评估方案

评估重点不是证明策略收益最高，而是验证系统是否能稳定、合理、可解释地完成目标任务。

预定义成功标准：

- 单股分析任务中，系统应输出明确建议、支持理由和至少一条风险提示。
- 候选股票筛选任务中，系统应能区分不同标的，而不是给出完全相同结论。
- 调仓任务中，系统不应违反预设持仓、现金或交易约束。
- 信息冲突或数据不完整时，系统应显式表达不确定性。
- 输出应包含可追溯的中间 Agent 结果。

Baseline 对比包括：

- 单 Agent 直接决策 baseline。
- 无风险管理版本。
- 只基于简单技术指标的规则版本。

## 7. 数据来源计划

项目计划使用小规模股票池和日频历史数据，避免高频交易或实时接入带来的额外复杂度。

计划数据源：

- 价格数据：Kaggle/S&P 500 Stocks 或本地 CSV。
- 新闻数据：Alpha Vantage News & Sentiment API，必要时补充 Finnhub company news。
- 基本面数据：SEC EDGAR Company Facts API 或 companyfacts.zip。

为保证课程复现，最终系统应保留离线缓存或样例数据，使没有 API key 时也能运行。
